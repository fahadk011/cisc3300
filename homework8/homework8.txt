2. Escaping a character means using a backslash (\) or other special character to 
ensure that characters with special meanings (like &, %, ?, etc.) are treated as 
literal characters, not as part of code or markup. This is common in URLs, JSON, 
HTML, and other contexts to avoid misinterpretation.

3. 200: OK - The request has succeeded.
400: Bad Request - The server could not understand the request due to invalid syntax.
401: Unauthorized - Authentication is required, or the provided credentials are not valid.
403: Forbidden - The server understood the request but refuses to authorize it.
404: Not Found - The requested resource could not be found on the server.
500: Internal Server Error - The server encountered an unexpected condition that prevented it from fulfilling the request.

4. Data validation is essential to ensure that the data received is accurate, 
secure, and in the correct format. It prevents issues such as SQL injection, 
cross-site scripting, and invalid data entry, which could lead to system errors, 
security vulnerabilities, or unexpected behavior.

5. Cookies are small pieces of data stored on the client-side, often used to keep 
track of user information across different pages or visits to a website.

Sessions are server-side storage of data that persists across multiple user 
interactions. Sessions are often used for storing user login information, 
preferences, or other temporary data. Both help manage state in web applications.
